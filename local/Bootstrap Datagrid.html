Bootstrap Datagrid

Minimalistic yet useful datagrid in Bootstrap style

Download this project as a .zip file Download this project as a tar.gz file

Simple datagrid table component that useful!

Bootstrap-Datagrid designed to be easily integrated with your bootstrap project. It exposes useful API that allow you to fully hook-in into the plugin

Markup

Switch regular table within your page into Bootstrap-Datagrid seamlessly by adding data-provide="datagrid" attribute

Code
<table class="table table-bordered" data-provide="datagrid">
  // ...
</table>
Result
# First Name Last Name Username
1 Mark Otto @mdo
Mark Otto @TwBootstrap
2 Jacob Thornton @fat
3 Larry the Bird @twitter

Usage

Beside using above data-attributes, you could call it via code

$("#some-table").datagrid()

Events/Hook

There are three main events within Bootstrap-Datagrid lifecycle : onBoot, onCommit and onClean. Heres an example to hook into those events via code:

Code
$("#target-table").datagrid({
  editable:true,
  onBoot: function(e){
    alert("Booting "
      +e.$table.prop("tagName").toLowerCase()
      +"#"
      +e.$table.attr("id")
      +" as Datagrid ...")
  },
  onCommit: function(e) {
    alert("Commiting '"+e.$editor.val()+"'...")
  },
  onClean: function(e) {
    alert("Clean triggered!")
  }
})

Inputs

By default, Bootstrap-Datagrid assume all editable cell(s) are text-based. However, its a breeze to support other input types. You can simply define your own input type and how you want it to interact with the overall cycle. Heres an example of custom inputs:

Code
// Html
<table class="table" id="custom-table">
  <thead>
    <tr>
      <th>#</th>
      <th>Item Name</th>
      <th>Item Category</th>
      <th>Price</th>
    </tr>
  </thead>
  <tbody>
    <tr class="active">
      <td data-editable="false">1</td>
      <td>Nike</td>
      <td data-type="select" data-value="shoe">Shoe</td>
      <td data-type="money" data-value="49">$49</td>
    </tr>
    //...
  </tbody>
</table>

// Script
$('#custom-table').datagrid({
  inputs: {
      select: {
        el : $('<select class="form-control datagrid-input">'),
        onShow:function(cell) {
          // Set the options
          if (!$(this).find('option').length) {
            $(this).append($('<option disabled="disabled">Select a category</option>'))
            $(this).append($('<option value="shoe">Shoe</option>'))
            $(this).append($('<option value="t-shirt">T-Shirt</option>'))
            $(this).append($('<option value="pants">Pants</option>'))
          }

          var inputPadding = parseInt(cell.data('padding'))-1
          $(this).css('padding', inputPadding+'px')
          $(this).css('width', '100%')
          $(this).css('height', '100%')
          $(this).css('top', cell.offset().top.toString+'px')
          $(this).css('left', cell.offset().left.toString+'px')

          $(this).val(cell.data('value'))
        },
        onChange:function(cell) {
          cell.data('value', $(this).val())
          cell.text($(this).find('option[value='+$(this).val()+']').text())
        },
        isChanged:function(cell) {
          return $(this).val() != cell.data('value')
        }
      },
      money: {
        el : $('<input type="text" class="form-control datagrid-input">'),
        onShow:function(cell) {
          var inputPadding = parseInt(cell.data('padding'))-1
          $(this).css('padding', inputPadding+'px')
          $(this).css('width', '100%')
          $(this).css('height', '100%')
          $(this).css('top', cell.offset().top.toString+'px')
          $(this).css('left', cell.offset().left.toString+'px')

          $(this).val(cell.data('value'))
        },
        onChange:function(cell) {
          cell.data('value', $(this).val())
          cell.text('$'+$(this).val())
        },
        isChanged:function(cell) {
          return $(this).val() != cell.data('value')
        }
      }
    }
  })

There are only 5 part(s) you need to concern about when you're defining your own input within inputs options.

  1. Input name obviously. In above example, we add two new input types, which are select and money. The default input type was text
  2. el (element) of your input. This parameter accepting regular jQuery selector as an argument.
  3. onShow event. Bootstrap-Datagrid will send cell element each time it constructing the input, so you can match your input aspect (offset, selected value, etc). Here you can also, for example, mutate the label with the value.
  4. isChanged event. Right before Bootstrap-Datagrid trying to commit and clean any changes on datagrid element, it will need to determine whether last active cell are changed or not. In this callback, you need to return bool value to tell the cell state
  5. onChange event. After Bootstrap-Datagrid know the state of the active cell, it will fired this event when it detect value changes. Here you can, for example, saving the data and/or formatting the cell text based by selected value.
NOTE

By default, all cell(s) are editable, but to disable some cell(s), all you have to do just setting data-editable to false within the td element (see above example).

Globally, you also can pass editable: false as part as datagrid option to construct the datagrid object without making all cell(s) instantly editable. Which you can enable later on using $('#some-table-selector').data('datagrid').boot()
